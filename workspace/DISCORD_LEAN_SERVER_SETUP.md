# DOODLES MASTER ACADEMY
## Lean Discord Launch Setup

This is the smallest version of the Discord server that is still safe, organized, and usable by one creator.

## 1. Server Purpose
- Run a structured art practice community around the first 5 foundational lessons.
- Give students a clear place to post each lesson.
- Keep feedback public, simple, and bounded.
- Support parents without turning the server into a general chat space.

Do not use this server as:
- a general social hangout
- an open-ended critique inbox
- a private DM-based teaching system

## 5. Launch Rules
Post these in `#rules-and-safety`.

1. Be kind. Encourage each other and keep comments respectful.
2. Keep feedback focused on the artwork, not the person.
3. No bullying, teasing, name-calling, or excluding others.
4. Do not share private information.
5. Use first names or nicknames only.
6. Do not message other students privately through the server.
7. Keep all art, language, and discussion kid-appropriate.
8. Lesson channels are for lesson submissions only.
9. Follow moderator and instructor directions.
10. Unsafe or repeated disruptive behavior can lead to removal.

## 7. Submission Standard
Use the same posting expectation in every lesson channel.

Students should post:
- 1 to 3 images
- one short reflection
- one main submission at a time

Recommended student caption format:
```md
Name: [first name or nickname]
What felt easy:
What felt hard:
What I want to improve:
```

## 8. Feedback System
At launch, keep critique selective.

### Default rule
- Students can encourage each other.
- Staff does not promise direct critique on every post.

### Instructor workflow
- Review posts once per week.
- Select a small number for direct reply.
- Keep replies public in the lesson channel.

### Critique formula
Use this every time:
1. What is working
2. Main fix
3. Next step

Example:
```md
What’s working: Your line confidence is already improving.
Main fix: Your longer strokes still wobble when you slow down too much.
Next step: Fill one page with shoulder-driven lines using the same length each time.
```

## 9. Recognition System
Launch with manual recognition only.

### Stars
- Award stars manually with a staff reaction.
- Use stars to reward:
  - completion
  - effort
  - improvement
  - creativity

### Hall of Fame
- Feature a few student posts each week or every other week.
- Do not feature only the most polished work.
- Include improvement picks, not just “best art.”

## 10. Moderation Policy

### Minor issues
- Remove the post if needed
- Redirect the student
- Give one clear warning

### Repeated issues
- Timeout
- Notify parent if relevant

### Serious issues
- Remove content immediately
- remove access or ban if needed
- document the incident in `#mod-room`

## 11. Minimal Bot Stack
Use one bot at launch.

### Recommended
- `Carl-bot` or `Dyno`

### Use it for
- automod
- logging
- basic moderation

### Avoid
- XP spam systems
- noisy welcome flows
- too many reaction role menus

## 12. Manual Launch Checklist
- Create the server and categories
- Create the 4 launch roles
- Apply channel permissions
- Post the pinned setup messages
- Install one moderation bot
- Enable automod for spam and unsafe links
- Turn on slowmode in `#studio-chat`
- Invite a small first cohort
- Run the first lesson submission cycle

## 13. First 30 Days Operating Rhythm
- Week 1: onboarding and Lesson 1 submissions
- Week 2: Lessons 2 and 3 start filling up
- Week 3: first Hall of Fame post
- Week 4: first light critique roundup

Keep the first month simple:
- no premium tier yet
- no complex badges yet
- no giant challenge calendar yet

The goal is to prove that the server stays calm, safe, and useful.
